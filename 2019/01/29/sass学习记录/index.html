什么是Sass以及安装

在说Sass之前，来说一下原始的CSS,你经常用到一个颜色，这时候你想要将其设置为一个变量，而CSS中原生变量定义语法是：--*,使用语法是var(--*);
写某个模块的时候，每次都要添加选择器的父选择器，是不是太麻烦了；
两个相似的模块，就是选择器名字不同，样式大体一致，我还要去再复制一遍吗。
上面的问题，Sass来帮你解决，sass其实本质来说还是CSS，只是专门的编程语言，写完再编译成正常的CSS文件，使用便利，节约时间，利于系统化管理与维护。

安装与使用

安装

npm install -g sass，说实话，命令行有点难用，懒得打字的直接下载转换应用=。=

使用

变量

SASS可以使用变量，所有变量以$开头。
如果变量需要在字符串中，就必须写在#{}之中。

计算

允许在代码中使用算式，其实css也有calc….

嵌套

选择器可以嵌套啦，以前那一长条选择器可以简化了，属性也可以嵌套，比如border下面有color,radius之类的，属性嵌套也要加:号，在嵌套的代码块，可以使用&引用父元素。

注释

标准的CSS注释 / comment / ，会保留到编译后的文件。
单行注释 // comment，只保留在SASS源文件中，编译后被省略。
在/*后面加一个感叹号，表示这是”重要注释”。即使是压缩模式编译，也会保留这行，通常可以用于声明版权信息。

继承

Mixin

使用@mixin定义一个代码块
使用@include调用这个代码块

插入文件

@import插入外部文件,

会发现为什么scss的文件实际都是有一个’_‘符号？因为@import根据文件名引入，在有些情况下，它会被编译为CSS的@import规则：
如果文件的扩展名是 .css。
如果文件名以 http:// 开头。
如果文件名是 url()。
如果 @import包含了任何媒体查询（media queries）。
如果上述情况都没有出现，并且扩展名是 .scss 或 .sass， 该名称的 Sass 或 SCSS 文件就被引入。 如果没有扩展名， Sass 将试着找出具有 .scss 或 .sass 扩展名的同名文件并将引入。

有scss或sass文件需要引入，又不希望编译为css文件，那么加一个下划线，就能避免编译，正常引入时也不需要加这个下划线。

注意，在同一个目录不能同时存在带下划线和不带下划线的同名文件。 例如，colors.scss 不能与 colors.scss 并存,添加下划线的文件会被忽略。

所以，实际运用情况是什么样的

首先，项目的css文件由Sass文件生成，Sass文件由各个模块组成
变量，mixin,框架，预设值放入base文件夹；
header,footer,section,nav放入layout文件夹；
blocks,buttons,components放入modules文件夹；
编辑的过程中，通过继承，mixin复用代码。

将需要的模块（外面的也行）按顺序@import成主scss文件，随后编译成css即可。
