ES6入门学习记录（三）

发表于 2019-01-28 | 分类于 FrontEnd ， 学习记录 | 阅读次数:

这部分比较关键，有Iterator和for..of循环，Generator函数，Promise对象，异步操作和Async函数，以及class.

Iterator和for…of循环

Iterator概念

1.为各种数据结狗提供一个统一简便的访问接口；2.使得数据结构的成员能够按某种次序排列；3.ES6创建了新的遍历命令for...of循环，Iterator主要供它消费。
遍历过程：

  1. 创建一个指针对象，指向当前数据结构的起始位置。
  2. 第一次调用指针对象的next方法，可以将指针指向数据结构的第一个成员。
  3. 不断调用next方法，直到指向数据结构的结束位置。

每一次调用next方法，都会返回数据结构的当前成员信息，包含value和done两个属性的对象。

数据结构默认Iterator接口

三类数据结构原生具备的：Array，Array-like Object,Set&Map.

调用Iterator接口的场合

  1. 结构赋值：进行结构赋值时，会默认调用Symbol.iterator方法。
  2. 扩展运算符：同上。
  3. yield*:跟一个可遍历结构，就会调用该结构遍历器接口。

for…of循环

for...of循环可以使用的范围包括数组、Set和Map结构、某些类似数组的对象（比如arguments对象、DOM NodeList对象）、Generator对象，以及字符串。

数组的遍历器接口

只会返回具有数字索引的属性

Generator函数

基本概念

Generator是ES6提供的一种异步变成解决方案。

  1. 从语法上可以将其理解为状态机，封装了多个内部状态；
  2. 执行generator函数会发挥一个遍历器对象，可以一次遍历内部每个状态；
  3. 形式上是一个普通函数，两个特征：一：function关键字和函数名之间有一个星号（*）；二：函数体内部使用yield语句，定义不同的内部状态。

调用Generator函数后，该函数并不会执行，返回的也不是函数运行结果，而是一个指向内部状态的指针对象，即遍历器对象。

yield语句

上面返回的遍历器对象，next方法运行逻辑：

  1. 遇到yield，暂停操作，将yield后面表达式的值，作为返回对象的value属性值。
  2. 没有遇到yield,一直运行到函数结束，直到return，返回return表达式(调用next才会执行求值)的值，作为value属性值。
  3. 如果没有return,返回对象value属性值为undefined.

注意：1.yield不能用在普通函数中；2.yield用在表达式中需要加括号。3.yield句本身没有返回值，next方法可以带一个参数，作为上一个yiled语句的返回值。

Generator.prototype.throw()

可以在函数体外抛出错误，在Generator函数体内捕获。
throw方法被捕获之后，会执行下一条yield语句。
Generator执行过程中抛出错误，没被内部捕获，就不会再执行下去，之后调用next方法，会返回{value:undefined，done:true}

Generator.prototype.return()

返回给定值，终结遍历。
如果内部有try...finally代码块，那么return会推迟到finally执行完再执行。

yield*语句

在generator函数内部调用另一个generator函数，默认情况下是没有效果的。
可以用yield 函数调用。 利用yield遍历二叉树

作为对象属性

generator的this

Generator函数不是普通构造函数，返回的总是遍历器对象，而不是this对象，这时候this为默认规则即指向window。
不能跟new一起用。

1.怎么绑在对象上；2.怎么执行new;
tips:prototype

generator与状态机

每运行一次改变一次状态。

generator与协程

协程（coroutine)，协作的线程或协作的函数。

  1. 协程与子例程的差异

    传统子例程采用堆栈式执行方式，只有子函数完全执行完毕，才结束执行父函数。可以并行执行，交换执行权的线程成为协程。

  2. 协程与普通线程的差异

    普通线程是抢先式，协程是自己分配。

应用

  1. 异步操作的同步化表达

  2. 控制流管理
    one by one可以按顺序排列，多个任务并列执行可以采取数组的写法。

  3. 部署iterator接口

  4. 作为数据结构

Promise

Promise新建后立即执行，然后then方法制定的回调函数将在当前脚本所有同步任务完成后执行。

异步操作和Async函数

先跳过generator函数和promise对象，弄清异步的问题方便理解generator和promise
ES6之前，异步变成方法：

  • 回调函数 //callback
  • 事件监听 //addEventListener
  • 发布/订阅
  • Promise对象

基本概念

class

class是一构造方法语法糖，其中定方法不需要加function关键字，方法之间也不需要逗号。
类方法的原型就是该类。
类原型的方法都是不可枚举的，ES5原型方法默认可枚举。
类属性名可采用表达式。

constructor方法

是类的默认方法，使用new命令生成对象实例时，自动调用该方法，默认返回实例对象this。

类的实例对象

生成类的实例都西昂需要使用new,不能像函数那样调用class.

Class表达式

用表达式定义时，类的名字是变量名,class后面的名字只在class内部可用，代指当前类（getClassName()可获取)。
采用表达式，可以写出立即执行的Class.

私有方法

一种做法是在命名前加区分，并不保险，只是约定而已。
第二是将私有方法移出模块，在模块内部在通过作用域绑定该方法。
第三是利用Symbol值的唯一性，将私有方法命名为一个Symbol值。

严格模式

类和模块的内部，默认就是严格模式。

name属性：函数的特性被Class继承，包括name属性

Class的继承

class sub extends sup {}，继承sup类的所有属性和方法。
super(arg)表示父类的构造函数，用来新建父类的this对象，子类必须在constructor里调用super方法，因为子类没有自己的this对象，而是继承父类this对象，对其加工。
如果子类没有定义constructor方法，就会被默认添加。

类的prototype属性和proto属性

Extends的继承目标

只要目标有prototype属性，就能被继承。
特殊情况：

  1. 子类继承Object类

就是构造函数Object的赋值，实例就是Object的实例

  1. 不存在任何继承

作为基类就是普通函数，直接继承Function.prototype,调用后返回一个对象，实例的原型对象的原型指向Object.prototype.

  1. 子类继承null
    类似第二种，原型对象原型指向undefined.

super关键字

  1. 作为函数调用，代表父类的构造函数；
  2. 作为对象调用，代表父类。

对象总是继承其他对象

实例的proto属性

子类的原型的原型===父类的原型

原生构造函数的继承

ES6为什么可以继承原生构造函数。（tips:this）
发现Object钩爪函数的行为，发现不是通过new Object()这种形式调用，规定忽略参数。

Class的gnerator方法

Class的静态方法

类中定义的方法，都会被实例继承。如果在一个方法前加上staric关键字，就表示该方法不会被实例继承，而是通过类来调用，称为“静态方法”。
可以被子类继承。

Class的实例和静态属性

  1. 类的实例属性
    类的实例属性可以用等式，写入类的定义之中。
  2. 类的静态属性
    同方法一样加上static.

new.target属性

返回new命令作用于的那个构造函数，如果构造函数不是通过new调用的，返回undefined。
子类继承父类时，会返回子类
