学习vue-cli的记录

安装

vue-cli已经3.0了， 请看官方文档
要使用2.0的vue init功能，需要全局安装桥接工具

npm i -g @vue/cli
3.0直接构建于webpack和webpack-dev-server之上。

初始化项目

vue create [projectname]

会自动在当前目录推导入口文件(main.js,index.js,App.vue,app.vue),还可以提供index.html,package.json，安装本地依赖，配置相应开发工具。

初始化文件说明

  1. .babelrc:babel的设置文件
  2. .editorconfig:编码格式配置文件
  3. .eslintignore:eslint忽略文件夹
  4. .gitignore:git不做处理文件
  5. .postcssrc.js:postcss的配置文件
  6. /build/:项目构建相关代码
  7. /config/:项目开发环境配置，开发环境变量，项目配置变量，生产环境变量
  8. /src/:源码目录，里面有vue公共组件，状态管理，页面入口文件，程序入口文件
  9. static:静态资源

我的理解：vue-cli基于webpack的功能搭建了一个初始化项目，预设了一些通用的配置。

插件和Preset

现有项目安装插件

开发模式

vue.config.js的devServer字段配置开发服务器

生产模式

–modern模式生成的包通过支持<script type="module>的浏览器中加载，旧版的包通过<script nomudule>加载，现代包解析和运算速度都会加快。

查看webpack config

vue-cli-service inspect
vue.config.js修改合并webpack配置。

HTML和静态资源

HTML

public/index.html被html-webpack-plugin模块处理。

不生成index.html,在vue.config.js中进行配置：

静态资源

  • 在js或template/CSS中通过相对路径被引用，会被Webpack处理，翻译为require()
  • 放在public目录下或通过绝对路径被引用，会被直接拷贝,不经过webpack处理。

    url转换规则

  • 绝对路径会被保留
  • .开头，会作为相对模块请求被解释并基于文件系统目录结构解析
  • ~开头，其后任何内容都会作为模块请求被解析
  • @开头，也会作为模块请求被解析，用处是Vue CLI会默认设置一个指向<projectRoot>/src的别名@，仅作用在模板中。

public文件夹

webpack打包的好处：

  • 脚本样式表全在一起，减少网络请求
  • 文件丢失在编译时报错，而不是客户端404
  • 生成文件都带hash，避免缓存旧版本

什么时候用public

  • 需要指定文件的名字；
  • 上千个图片，需要动态引用路径；
  • 有些库与Webpack不兼容，只能使用独立的script标签引入。

webpack相关

最简单的配置方式

在vue.config.js的configureWebpack提供一个webpack配置对象，会被merge。
基于环境配置或者想直接修改可以设置为一个函数，第一个参数会收到解析好的config。在函数内就可以修改配置或者返回一个将被merge的对象。
process.env.NODE_ENV是环境变量。

链式操作

vue-cli是通过webpack-chain维护的webpack，具体操作看webpack-chain。

部署

项目部署到github:

  1. 在vue.config.js设置正确的publicpath:
    部署到仓库，将publicPath设为/<repo>/.

  2. 项目目录下，建立deploy.sh运行进行部署

配置：
vue.config.js

  1. publicPath:应用包的基本URL,默认为域名的根路径，子路径需要添加相应字段，也可以设置为空字符串和相对路径，所有打包资源会被连接到相对路径，开发时可以判断process.env.NODE_ENV来实现多模式。
  2. outputDir: 生产目录
  3. assetsDir: 静态资源目录(相对于outputDir)
  4. indexPath: 生成的index.html的输出路径(相对于outputDir)
  5. filenameHashing: 文件是否要hash
  6. pages: 多页模式构建应用。每个page都有对应的js入口文件。值是对象，key是入口名字，value是：指定entry,template,filename,title和chunks的对象，或者entry字符串。
  7. lintOnSave: 每次保存时lint代码。
  8. runtimeCompiler: 使用包含运行时编译器的Vue构建版本。Vue组件中使用template选项。
  9. productionSourceMap: 生产环境的source map,不需要就关闭。
  10. crossorigin: 生成HTML中link和script标签的crossorigin属性,anonymous和use-credentials。
  11. intergrity: 生成的HTML中link和script标签启动SRI。如果是部署在CDN,提供安全性。
  12. configureWebpack: webpack配置对象。
  13. chainWebpack: 基于Webpack-chain的实例，细致修改。
  14. css.extract: 将组件的CSS提取到独立的CSS文件中。
  15. css.sourceMap: 为CSS开启吗，默认关。
  16. css.loaderOptions: 对象，像loader传递参数。
  17. devServer: 所有webpack-dev-server配置选项。
  18. devServer.proxy: 指向开发环境API请求代理到API服务器，将任何未知请求代理到该项设置(字符串)，多项情况是对象{‘api’:{target:’‘}}。
  19. pluginOptions: 传递第三方插件选项。

其中configureWebpack,chainWebpack,devServer,devServer.proxy可能会比较容易用到。
