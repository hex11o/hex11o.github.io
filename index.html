关于中文翻译的吐槽

发表于 2019-03-05 | 分类于 杂谈 | 阅读次数:

事情起因，群里有人发图：


那位朋友说规则第三点测试不对，想问解释。
然后我测试了一下,内部元素的margin box左边肯定是不会挤掉BFC的padding的，更不可能串到border-box的左边去。(你transform倒是可以=.=)
我就去看了W3C的原文：

In a block formatting context, each box’s left outer edge touches the left edge of the containing block (for right-to-left formatting, right edges touch).

哪里有说border-box?只是提到内部元素盒子会靠到包含块(containing block)的左边,并没有说靠在包含块的border-box上啊。
继续看W3C对containing box的定义：

The position and size of an element’s box(es) are sometimes calculated relative to a certain rectangle, called the containing block of the element.

我翻译一下：元素盒子的位置和大小又是是通过特定的方形确定的，称为元素的包含块。

关键是：这个包含块是内部元素的,而并不是BFC的。

下面简单说下包含块的方形由什么决定，仔细看的可以点上面的链接看原文：

  1. 根元素(html)的包含块是视窗的框框;
  2. 如果元素的position是relative或者static,那么包含块就是最近祖元素的content edge(这个不知道是什么的去看盒模型);
  3. 如果元素的position是fixed,包含块是视窗的框框;
  4. 如果元素的position是absolute，那么包含块是通过position不为static的最近祖元素按以下规则继续判断：
    1. 如果祖元素是行内元素，那么包含块就是行内元素的padding box的bounding box，如果行内元素跨越多行，那么包含块就是undefined;
    2. 其他情况：包含块都是祖元素的padding edge。

有什么问题，直接去看W3C的定义，中文语境太模糊了，翻译水平差的坑越带越深。

Difference between querySelectorAll and getElementBy[]

发表于 2019-03-03 | 分类于 FrontEnd ， JavaScript | 阅读次数:

用querySelectorAll和getElementBy[]时发现它们之间的差别：
querySelectorAll选出的是Nodelist,getElementBy选出来的是HTMLCollection;

  1. Nodelist: NodeList某些情况下是动态的，比如childNodes,进行更新后会反映到NodeList中，其他情况下都是静态的。
  • NodeList的方法：item(),entries(),forEach(),keys(),values()
  • NodeList的属性: length
  1. HTMLCollection: 是类数组的数据，它会根据文档变化进行动态更新，所以你对节点进行遍历操作删减选择器的话,可能出现只操作一半的情况。
  • HTMLCollection的方法: item(),namedItem()
  • HTMLCollection的属性: length

vuex学习记录

发表于 2019-02-26 | 阅读次数:

安装

项目文件夹下安装：
npm i --save vuex

基本使用

Vuex仅仅包含一个store实例。

state

Vue组件中获得Vuex状态：

  1. 计算属性
    store.state.count
    问题：依赖全局状态单例。在每个需要使用state的组件中都需要频繁导入。
  2. 组件注入
    通过store属性，将根组件注入到子组件中，子组件通过this.$store访问。
  3. mapState辅助函数
    组件需要获取多个状态，单个赋值太麻烦，可以使用辅助函数来生成计算属性。
  • 箭头函数
  • 字符串
  • 局部状态（this）必需使用常规函数,箭头函数的this指向
  • 字符串数组属性
  • 对象展开运算：辅助函数返回的是对象，与局部计算属性混合

getters

需求情况，对store中的state派生一些状态，进行多次复用，相当于vue的computed对data的处理。
getters能接收其他getters作为第二个参数。
暴露为store.getters对象。
getters能返回函数实现给getter传参。
mapGetters辅助函数.

mutions

更改store中state的唯一方法就是提交mutation, 每个mutation都有一个事件类型和一个回调函数。
要调用事件，需要使用store.commit方法。
可以传入额外参数。

  1. 提前初始化好所需属性
  2. 添加新书性，应该使用Vue.set或者以新对象替换老对象,(扩展对象)。
  3. mutation必需是同步函数，因为mutation触发时，回调函数还没有被调用。
  4. mapMutations()将this.$store.commit('mutation')映射为this.mutation(),支持载荷.

actions

action函数接收一个与store实例具有相同方法和属性的context对象，所以我可以调用context.commit提交mutation以及获取state和getters.
actions通过store.dispatch触发.
store.dispatch可以处理返回的promise,返回的也是promise.

modules

单一状态树，store对象会变得特别臃肿。
利用模块对store进行分割,嵌套。

模块的内部状态

  1. 内部的mutation和getter，第一个参数是模块局部状态对象。
  2. 内部的action, 局部状态state通过context.state暴露出来，根节点为context.rootstate.

命名空间

默认情况，模块内部的action，mutation和getter是注册在全局命名空间的，这样多个模块能对同一mutation和action做反应。

  1. 可以添加namespaced:true使其成为带命名空间的模块,注册之后，所有getter,actions,mutation都会自动根据模块注册的路径调整命名。

    注意，state本身就是嵌套的。

  2. 嵌套模块继承父模块的命名空间。

带命名空间的模块访问全局内容

通过rootState和rootGetter作为第三四个参数传入getter,同时通过context对象传入action。
