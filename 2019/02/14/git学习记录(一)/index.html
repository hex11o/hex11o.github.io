Git的存储方式

git通过对文件生成快照，提交更新时，如果文件快照没有修改，则只保留一个链接指向之前存储的文件。

校验机制为SHA-1散列（hash）————由40个十六制字符串构成。

Git三种状态(committed||modified||staged)

项目的三个工作区域(工作目录，暂存区域，Git仓库)

  1. 工作目录：对项目某个版本独立提取出来的内容
  2. 暂存区域：保存下次将提交的文件列表信息；
  3. Git仓库：保存项目的元数据和对象数据库的地方。

  2. format, 定制要显示的记录格式

    %s 提交说明

Git基础–远程仓库

  1. git remote -v //显示读写远程仓库使用的git保存的简写与对应的URL

  2. 添加远程仓库
    git remote add [shortname] [url]

  3. 抓取远程仓库//抓取并不会自动合并
    git fetch [shortname]||[url]
  4. 拉取远程仓库//自动尝试合并到当前所在分支
    git pull [remote-name]
  5. 推送到远程仓库
    git push [remote-name] [branch-name]
  6. 查看远程仓库
    git remote show [remote-name]
