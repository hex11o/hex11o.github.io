npm使用说明

发表于 2019-01-29 | 分类于 FrontEnd ， 工具 | 阅读次数:

本文是对npm学习的归纳。
在安装完node后自带的npm包管理器的使用，会用一些简单功能，看看官方文档更深入一点了解npm.

简单说点别的：vi

在看npm视频时发现作者用了个vi命令，能看出新建了个文件并可以打开编辑，于是我好奇查了一下vi是什么，原来是Unix编辑器，这里简单说一下，具体操作见编辑过程链接，不在此赘述：

开始

vi filename.filetype新建一个文件并打开，编辑时有两个模式，一个是命令模式，一个是输入模式。

  1. 命令模式：默认模式，也就是打开文档就处于这个模式，命令是大小写敏感的，大多数命令输入时是不会显示在屏幕上的，有些命令会显示在最后一行：：/？
  2. 输入模式：文本创建的模式，返回命令模式就按Esc.

编辑过程

具体操作及命令说明可参见Using vi

退出

在命令模式下

npm

NPM的locally和globally

如果只是自己模块需要，比如require，那么就使用locally。npm install pkgname
如果是在cmd里使用，那就使用globally。npm install -g pkgname

通过package.json安装依赖包

如果文件夹里有package.json文件，在命令行输入npm install，那么npm会找寻package.json中的依赖包，并将其全部下载。

package.json

该文件有两个必需键值：name和version。
添加node中已有的安装包到package.json的dependicies，使用npm install packagename --save

卸载全局安装包

npm ls -g --depth=0查看全局安装的包；
npm uninstall -g packagename卸载该包。

创建Node.js模块

自动创建package.json

npm init：name,version,description,entry point,test command,git repository,keywords,author,license.
npm publish

发布npm包

npm adduser:添加用户
npm config ls:查看npm配置
npm publish：发布包

升级包并发布

更改后发布需要更改版本号，可以直接在package.json修改，或者使用npm version patch生成。然后npm publish.
版本号的修改规则：
第一位：改变api，不往前兼容，major release;
第二位：新特性，向前兼容，minor release;
第三位：简单的bug修复，没有新特性,patch release.
对应的版本修改就是npm version major,npm version minor,npm version patch.
