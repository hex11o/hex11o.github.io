Iterator和for…of循环

Iterator概念

1.为各种数据结狗提供一个统一简便的访问接口；2.使得数据结构的成员能够按某种次序排列；3.ES6创建了新的遍历命令for...of循环，Iterator主要供它消费。
遍历过程：

  1. 创建一个指针对象，指向当前数据结构的起始位置。
  2. 第一次调用指针对象的next方法，可以将指针指向数据结构的第一个成员。
  3. 不断调用next方法，直到指向数据结构的结束位置。

每一次调用next方法，都会返回数据结构的当前成员信息，包含value和done两个属性的对象。

数据结构默认Iterator接口

三类数据结构原生具备的：Array，Array-like Object,Set&Map.

调用Iterator接口的场合

  1. 结构赋值：进行结构赋值时，会默认调用Symbol.iterator方法。
  2. 扩展运算符：同上。
  3. yield*:跟一个可遍历结构，就会调用该结构遍历器接口。

for…of循环

for...of循环可以使用的范围包括数组、Set和Map结构、某些类似数组的对象（比如arguments对象、DOM NodeList对象）、Generator对象，以及字符串。

数组的遍历器接口

只会返回具有数字索引的属性

Generator函数

基本概念

Generator是ES6提供的一种异步变成解决方案。

  1. 从语法上可以将其理解为状态机，封装了多个内部状态；
  2. 执行generator函数会发挥一个遍历器对象，可以一次遍历内部每个状态；
  3. 形式上是一个普通函数，两个特征：一：function关键字和函数名之间有一个星号（*）；二：函数体内部使用yield语句，定义不同的内部状态。

调用Generator函数后，该函数并不会执行，返回的也不是函数运行结果，而是一个指向内部状态的指针对象，即遍历器对象。

yield语句

上面返回的遍历器对象，next方法运行逻辑：

  1. 遇到yield，暂停操作，将yield后面表达式的值，作为返回对象的value属性值。
  2. 没有遇到yield,一直运行到函数结束，直到return，返回return表达式(调用next才会执行求值)的值，作为value属性值。
  3. 如果没有return,返回对象value属性值为undefined.

注意：1.yield不能用在普通函数中；2.yield用在表达式中需要加括号。3.yield句本身没有返回值，next方法可以带一个参数，作为上一个yiled语句的返回值。

Generator.prototype.throw()

可以在函数体外抛出错误，在Generator函数体内捕获。
throw方法被捕获之后，会执行下一条yield语句。
Generator执行过程中抛出错误，没被内部捕获，就不会再执行下去，之后调用next方法，会返回{value:undefined，done:true}

Generator.prototype.return()

返回给定值，终结遍历。
如果内部有try...finally代码块，那么return会推迟到finally执行完再执行。

yield*语句

在generator函数内部调用另一个generator函数，默认情况下是没有效果的。
可以用yield 函数调用。 利用yield遍历二叉树

作为对象属性

generator的this

Generator函数不是普通构造函数，返回的总是遍历器对象，而不是this对象，这时候this为默认规则即指向window。

generator与协程

协程（coroutine)，协作的线程或协作的函数。

  1. 协程与子例程的差异

    传统子例程采用堆栈式执行方式，只有子函数完全执行完毕，才结束执行父函数。可以并行执行，交换执行权的线程成为协程。

  2. 协程与普通线程的差异

    普通线程是抢先式，协程是自己分配。

Proxy

用于修改某些操作的默认行为，相当于在目标对象前假设一层操作，必须通过这层操作，就是代理，翻墙的懂的=。=
使用方法：new Proxy(target,handler).
支持的拦截操作一览：

  1. get(target,propKey,receiver)
    代理对象的属性读取，最后一个参数receiver是一个对象，可选。
  2. set(target,propKey,value,receiver)
    代理对象属性的设置，返回一个布尔值。
  3. has(target,propKey)
    代理propKey in proxy的操作，以及hasOwnProperty的方法，返回布尔值。
  4. deleteProperty(target,propKey)
    代理delete proxy[propKey]的操作返回布尔值。
  5. ownKeys(target)
    代理Object.getOwnPropertyNames(proxy) 、 Object.getOwnPropertySymbols(proxy) 、Object.keys(proxy) ，返回一个数组。该方法返回对象所有自身的属性，而 Object.keys() 仅返回对象可遍历的属性。
  6. getOwnPropertyDescriptor(target, propKey)
    拦截 Object.getOwnPropertyDescriptor(proxy, propKey)，返回属性的描述对象。
  7. defineProperty(target, propKey, propDesc)
    拦截 Object.defineProperty(proxy, propKey,propDesc） 、 Object.defineProperties(proxy, propDescs) ，返回一个布尔值。
  8. preventExtensions(target)
    拦截 Object.preventExtensions(proxy) ，返回一个布尔值。
  9. getPrototypeOf(target)
    拦截 Object.getPrototypeOf(proxy) ，返回一个对象。
  10. isExtensible(target)
    拦截 Object.isExtensible(proxy) ，返回一个布尔值。
  11. setPrototypeOf(target, proto)
    拦截 Object.setPrototypeOf(proxy, proto) ，返回一个布尔值。
    如果目标对象是函数，那么还有两种额外操作可以拦截。
  12. apply(target, object, args)
    拦截Proxy实例作为函数调用的操作，比如 proxy(...args) 、 proxy.call(object,...args) 、 proxy.apply(...) 。
  13. construct(target, args)
    拦截Proxy实例作为构造函数调用的操作，比如 new proxy(...args) 。

Reflect概述

  1. 将Object对象的一些明显属于语言内部的方法，放在Reflect对象上。
  2. 修改某些Object方法的返回结果，让其变得更合理。比如， Object.defineProperty(obj, name, desc) 在无法定义属性时，会抛出一个错误，而 Reflect.defineProperty(obj, name, desc) 则会返回 false 。
  3. 让Object操作变成函数行为。某些 Object 操作是命令式，比如 name in obj 和 delete obj[name]，而 Reflect.has(obj,name) 和 Reflect.deleteProperty(obj, name)让它们变成了函数行为。
  4. Reflect 对象的方法与 Proxy 对象的方法一一对应，只要是 Proxy 对象的方法，就能在 Reflect 对象上找到对应的方法。这就让 Proxy 对象可以方便地调用对应的 Reflect 方法，完成默认行为，作为修改行为的基础。也就是说，不管 Proxy 怎么修改默认行为，你总可以在Reflect 上获取默认行为。

二进制数组

WebGL：浏览器与显卡之间通信接口，必须是二进制的，不然都要进行格式化会相对耗时。
二进制数组由三类对象组成：

  1. ArrayBuffer对象：代表内存中有段二进制数据，可以通过“视图”进行操作。
  2. TypedArray视图：共包括9种类型的视图，比如Uint8Array(无符号8位),Int16Array(16位整数),Float32Array(32位浮点数).
  3. DataView视图：自定义符合格式的视图，比如第一个字节是什么类型，第二三个是什么类型。

简单说，ArrayBuffer对象代表原始的二进制数据，TypedArray视图用来读写简单类型的二进制数据，DataView视图用来读写复杂类型的二进制数据。

ArrayBuffer对象

ArrayBUffer代表存储二进制数据的一段内存，不能直接读写，只能通过视图来读写。
TypedArray视图的构造函数，除了接受 ArrayBuffer 实例作为参数，还可以接受普通数组作为参数，直接分配内存生成底层的ArrayBuffer实例，并同时完成对这段内存的赋值。
ArrayBuffer.prototype.byteLength:返回内存区域的字节长度。
ArrayBuffer.prototype.slice():将内存区域的一部分，拷贝生成一个新的ArrayBuffer对象，分配新内存–拷贝。
ArrayBuffer.isView():表示参数是否为ArrayBuffer的视图实例。

TypedArray视图

  1. TypedArray(buffer, byteOffset=0, length?):操作ArrayBuffer对象,开始位置，数据长度。
  2. TypedArray(length):直接分配内存生成。
  3. TypedArray(typedArray):接受另一个实例作为参数，赋值构建视图。
  4. TypedArray(arrayLikeObject):普通数组变实例。

数组方法

普通数组的方法和属性适用于TypedArrray,但是没有concat方法。

字节序

指的是数值在内存中的表现方式。
x86体系的计算机都采用小端字节序（little endian），相对重要的字节排在后面的内存地址，相对不重要字节排在前面的内存地址。比如，一个占据四个字节的16进制数 0x12345678 ，决定其大小的最重要的字节是“12”，最不重要的是“78”。小端字节序将最不重要的字节排在前面，储存顺序就是 78563412。

BYTES_PER_ELEMENT属性：返回数据类型占据的字节数，实例也能获取。

字符串与ArrayBuffer的互相转换

溢出

处理规则：1.正向溢出：最小值加上余值，再减1；2.反向溢出：最大值减去余值，再加上1。
Uint8ClampedArray规则则是大小顶边。

TypedArray.prototype.buffer：返回ArrayBuffer对象，只读。

TypedArray.prototype.byteLength，TypedArray.prototype.byteOffset，TypedArray.prototype.length

TypedArray.prototype.set()：赋值数组，整段内存赋值，第二个参数表示从对象第几个成员开始赋值参数对象。
TypedArray.prototype.subarray():参数为起始，返回一个新视图。
TypedArray.prototype.slice()：参数为开始位置，可以为负数。返回一个新实例。
TypedArray.of():将参数转为一个TypedArray实例。
TypedArray.from():将可遍历数据解构转为一个TypedArray实例，接受第二个参数对每个元素进行遍历，类似map方法，先拷贝再处理。

复合视图

同一段内存，可以存放不同类型的数据。

DataView视图

ArrayBuffer 对象的各种TypedArray视图，是用来向网卡、声卡之类的本机设备传送数据，所以使用本机的字节序就可以了；而 DataView 视图的设计目的，是用来处理网络设备传来的数据，所以大端字节序或小端字节序是可以自行设定的。
DataView提供八个方法读取内存：
getInt8,getUint8,getInt16,getUint16,getInt32,getUint32,getFloat32,getFloat64.
默认使用大端字节序解读数据，如果需要使用小段字节序，必须在get第二个参数使用true.
DataView提供八个方法写入内存：
set...同上，第一额参数是字节序号，第二个参数为写入数据，第三个参数为写入字节的方法，大端还是小端，同上。

Map

JS允许使用\uxxxx表示一个字符，那非plane 0的会理解为“\uxxxx+X”,用大括号{}包含码点（\u{xxxxx}）就可以正确解读，。

codePointAt()
JS中，字符以UTF-16的格式存储，每个字符固定为2Bit,对于非零面的字符，就会被认为是两个字符。
charCodeAt只能判断为两个字符；
codePointAt能正确处理四个字节储存的字符，但下一个字符还是可以返回码点，即四个字节返回一个字符，三四字节又返回一个。

用for..of..可以不返回三四字节的那个字符，即遍历器。

String.fromCodePoint()
fromCharCode不能识别非零面，最高位被舍弃。
fromCodePoint就可以弥补，作用与codePointAt()相反，一个是字符变码点，一个是码点变字符。

at():识别非零面字符，返回正确字符。

normalize():接受一个参数制定normalize的方式，合成，分解（欧洲语言）。

includes():返回布尔值，表示是否找到参数字符串。
startsWith():返回布尔值，是否在源字符串的头部。
endsWith():返回布尔值，是否在源字符串的尾部。
*以上三个方法都至此第二个参数，表示开始搜索的位置。

repeat(n):返回一个新字符串，表示源字符串重复n次,参数是效数会被取整，负数(<-1)或Infinity报错,NaN，(-1<n<1)等于0。
padStart(n,str),padEnd(n,str):字符串补全，第一个参数为字符串最小长度，第二个参数是用来补全的字符串，没有第二个参数就会用空格补全。

模板字符串

模板字符串（template string）是增强版字符串，用反引号（`）标识。它可以当作普通字符串使用，也可以用来定义多行字符串，或者在字符串嵌入变量。
嵌入变量需要将变量名写在${}中，可以进行运算，可以调用函数，大括号内的值如果不是字符串，将调用toString方法。
模板字符串还可以嵌套。

标签模板

模板字符串可以紧跟在一个函数名后面，该函数就会被调用来处理这个模板字符串。
如果模板字符里面有变量，会将模板字符串先处理成多个参数，再调用函数。

可以利用这些参数做一些事情，比如在JS里运行其它的代码。

String.raw()：返回一个斜杠都被转义的字符串，如果已经被转义那么不会做任何处理。我测试chrome并没有改变，先不管
可以做正常函数使用，第一个参数应该是具有raw属性的对象，值应该是一个数组。

正则的扩展

构造函数会忽略原有的正则表达式的修饰符，使用新指定修饰符。

字符串额正则方法

字符串有四个方法，可以使用正则表达式：match(),replace(),search(),split().
都是调用的RegExp.prototype[Symbol.xxxx]

u修饰符

正确处理非零面字符。

  1. 点字符
    对于非零面字符，正则的点字符不能识别，必需加上u修饰符。
  2. 字符表示法
    大括号也必需加上u修饰符才能识别,不加就会被认为匹配61个连续u。
  3. 量词
    量词识别与大括号解读unicode的区分
  4. 预定义模式
    反正就是识别大于0xFFFF的字符。
  5. i修饰符
    编码不同，字型相近，加u才能识别非规范字符。

y修饰符

与g相似，蛋下次搜索必须在lastIndex位置发现匹配。
sticky属性：表示是否设置了y修饰符；
flags属性：返回正则表达式的修饰符；
source属性：返回正则表达式的正文；
RegExp.escape():将字符串进行转义，str.replace(/[\-\[\]\/\{\}\(\)\*\+\?\.\\\^\$\|]/g, '\\$&')

后行断言

/(?<=y)x/:x在y后面才匹配
/(?<!y)x/：x不在y后面才匹配

数组的扩展

Array.from()
可以将类似数组的对象和可便利额对象转换为真正的数组,ES5中实现是[].slice.call(obj).
Array.from()接受第二个参数，类似于map方法，对每个元素进行处理，处理后的值放入返回的数组。

扩展运算符...也可以将某些具有遍历器接口的对象转为数组。

Array.of()：用于将一组值转换为数组，Array()构造函数由于参数个数不同，行为会有差异，参数不少于两个时，才会返回有参数组成的新数组。
copyWithin()
将指定位置成员赋值到其他位置，返回当前数组：

  • target（必需）:从该位置开始替换数据；
  • start（可选）：从该位置开始读取数据，默认为0。负数表示倒数。
  • end(可选)：到该位置前停止读取，默认数组长度。负数表示倒数。

find()和finIndex()
find找出第一个符合条件的数组成员，没有就返回undefined，回调函数可以接受三个参数，依次是当前值，当前位置和原数组，第二个参数绑定回调函数的this对象。
findIndex返回第一个位置，没有就返回-1，可以发现NaN，借助Object.is().
fill()：填充数组，参数分别是填充值，开始，结束位置。
entries(),keys(),values()
都返回一个遍历器对象，可以用for...of进行遍历，key()是键名的遍历，values()是键值的遍历，entries()是键值对的遍历。

includes():
是否包含给定值，第二个参数是搜索位置；负数则倒数，负数长度大于数组长度则会重置为0；
之前是使用indexOf()，首先是不够语义化，二是内部使用严格相当运算符（===），会导致对NaN的误判，includes就可以进行判断。

Map和Set数据结构有一个has方法，需要与includes区分：

  • Map结构的has方法，是用来查找键名的。
  • Set结构的has方法，是用来查找值的。

数组的空位

空位不是undefined
ES5对空位的处理：

  • forEach(),filter(),every()和some()都会跳过空位。
  • map()会跳过空位，但为保留这个值
  • join()和toString()会将空位视为undefined,而undefined和null会被处理为空字符串。

ES6明确将空位转为undefined

entries() 、 keys() 、 values()、 find() 和 findIndex() 会将空位处理成undefined。

箭头函数

箭头函数代码块多于一条语句，就要用大括号括起来，并用return返回。
大括号被解释为代码块，如果箭头函数直接返回一个对象，必需在对象外加上括号。
箭头函数可以简化回调函数。
注意点：

  1. this对象，箭头函数本身没有this,导致内部的this就是外层代码块的this。
  2. 不能当作构造函数，就是说不能使用new操作符（因为没有this)。
  3. 不能使用arguments对象，要用就用Rest参数。
  4. 不可以使用yield命令，所以不能做Generator函数。
  5. 除开this，还有三个变量也是指向外层函数的对应变量，arguments, super, new.target.
  6. λ演算.

函数绑定

函数绑定运算符是两个双冒号(::),双冒号左边是对象，右边是函数，就把对象做上下文环境（this），绑在函数上。
双冒号左边为空，右边是一个对象的方法，则等于把该方法绑定在该对象上。

尾调用（Tail Call）优化

尾调用是指函数最后一步操作是直接调用另一个函数。
函数调用会在内存形成一个调用记录，又称“调用帧”（call frame）,保存调用位置和内部变量等信息，所有的调用帧会形成一个“调用栈”（call stack）。

尾递归

尾调用自身就成为尾递归，递归很容易发生栈溢出的错误。但对于尾递归来说，只存在一个调用栈，不会发生栈溢出错误。

递归函数的改写

就是把所有用到的内部变量改写成函数的参数。
柯里化(curring)：将多参数函数转换为单参数的形式
函数默认值

严格模式

尾调用优化只在严格模式下开启，因为正常模式下，有两个变量跟踪函数的调用栈。

  • func.arguments:返回调用时函数的参数；
  • func.caller:返回调用当前函数的那个函数。

严格模式禁用这两个变量。

正常模式如何使用尾递归优化

采用循环换掉递归

对象的扩展

Object.is()
ES5比较两个值只有两个运算符：相等运算符（==）和严格相等运算符（===）。
前者会自动转换数据类型，后者NaN不等于自身，以及+0=-0。
Object.is(a,b)类似于===，ES5实现is:

Object.assign()
用域对象横，将源对象的所有可枚举属性，赋值到目标对象,同名属性会被覆盖。
Object.assign(targer,source1,source2);
只有一个参数，会转换为对象，所以undefined和null会报错。
非对象其他类型的值（即数值、字符串和布尔值）不在首参数，都不会报错。但是，除了字符串会以数组形式，拷贝入目标对象，其他值都不会产生效果。

Object.assign是浅拷贝。

常见用途：

  1. 为对象添加属性；
  2. 为对象添加方法；
  3. 克隆对象；
  4. 合并多个对象；
  5. 为属性指定默认值；

属性的可枚举性

ES5:

  • for..in：遍历自身和继承可枚举；
  • Object.keys():自身可枚举；
  • JSON.stringify()：串行化自身可枚举；
    ES6:
  • Object.assign():拷贝自身可枚举；

所有class的原型哦都是不可枚举的

属性的遍历

  1. for..in
  2. Object.keys(obj)：返回所有可枚举属性（不包括Symbol）
  3. Object.getOwnPropertyNames(obj)：返回自身所有属性，只是不包括Symbol属性
  4. Object.getOWnPropertySymbols(obj)：返回所有自身Symbol属性
  5. Reflect.ownKeys(obj)：返回所有自身属性。

按相同的属性遍历次序规则：

  • 先遍历属性名为数值的属性，按数字排序；
  • 再遍历属性名为字符串的属性，按生成时间排序；
  • 再遍历所有属性名为Symbol值的属性，生成时间排序。

对象扩展运算符

Rest解构赋值

要求等号右边是一个对象，必须是最后一个参数，浅拷贝，取得是等号右边所有尚未读取的键。
不会拷贝继承自原型对象的属性，（正常解构会查到原型链上去）。

扩展运算符

取出参数对象的所有可遍历属性，拷贝到当前对象中。
用户自定义属性放在扩展运算符后面，则扩展运算符内部同名属性会被覆盖，用来修改现有对象部分的部分属性就很方便。
自定义放前面，就是默认属性值。
扩展运算符的参数对象中，如果有取值函数get,这个函数会被执行。
Object.getOwnPropertyDescriptors:
返回一个对象，所有元对象的属性名和属性值。
解决Object.assign()无法拷贝get属性和set属性的问题。

Symbol

保证每个属性的名字都是独一无二，这样就从根本上防止属性名的冲突。这就是ES6引入Symbol的原因。
Symbol值不能用new,因为它不是对象。
Symbol可以接受一个字符串作为参数，表示对Symbol实例的描述。
Symbol不能进行计算但是可以转换为字符串，布尔值。
使用Symbol时，需要放在方括号中。
symbol.for()和Symbol.keyFor()
接受一个字符串为参数，在全局环境搜索以该参数作为名称（无参数则生成Symbol(undefined)）的Symbol值，如果有就返回该值，没有就创建（搜索不到Symbnol()创建值）。

内置的Symbol值

Symbol.hasInstance

对象的属性，指向内部方法，当其他对象使用instanceof运算符时，调用这个的方法。

Symbol.isConcantSpreadable

对象的Symbol.isConcantSpreadable等于一个布尔值，没有的时候是Undefined，表示该对象使用Array.prototype.concat()时，是否可以展开。
类数组对象也可以展开，但该属性默认为false,需手动打开。

Symbol.species

指向一个方法。对象作为构造函数创建实例时，会调用该方法。如果this.constructor[Symbol.species]存在，就会使用这个属性作为构造函数，来创造新的实例函数。

Symbol.match

对象的 Symbol.match 属性，指向一个函数。当执行 str.match(myObject) 时，如果该属性存在，会调用它，返回该方法的返回值。

Symbol.replace

对象的 Symbol.replace 属性，指向一个方法，当该对象被 String.prototype.replace 方法调用时，会返回该方法的返回值。

Symbol.search

对象的 Symbol.search 属性，指向一个方法，当该对象被 String.prototype.search 方法调用时，会返回该方法的返回值。

Symbol.split

对象的 Symbol.split 属性，指向一个方法，当该对象被 String.prototype.split 方法调用时，会返回该方法的返回值。

Symbol.iterator

对象的 Symbol.iterator 属性，指向该对象的默认遍历器方法。

Symbol.toPrimitive

对象的 Symbol.toPrimitive 属性，指向一个方法。该对象被转为原始类型的值时，会调用这个方法，返回该对象对应的原始类型值。

Symbol.toStringTag

对象的 Symbol.toStringTag 属性，指向一个方法。在该对象上面调用 Object.prototype.toString 方法时，如果这个属性存在，它的返回值会出现在 toString 方法返回的字符串之中，表示对象的类型。也就是说，这个属性可以用来定制 [object Object] 或 [object Array] 中object后面的那个字符串。

Symbol.unscopables

对象的 Symbol.unscopables 属性，指向一个对象。该对象指定了使用 with 关键字时，哪些属性会被 with 环境排除。
