You dont konw JS学习笔记(一)

发表于 2019-01-21 | 分类于 FrontEnd ， JavaScript | 阅读次数:

看了大半个月的《Javascript高级程序设计》（以下简称高程），把百度学院的前端课程学到一堆设计模式时脑袋开始有点转不过来了，重新开始看书，试图了解为什么要那么做的原因。其实一开始看高程时应该要配合you dont konw JS这本书一起看的，因为初学是真的不懂JS啊, 其实高程虽然内容都提到了，但是我看高程很难抓到脉络=.=
推荐一个工具，可以看到代码一步步怎么运行的——Visualize Code.
因为看完书很容易忘记，写一些读书笔记整理思路复习一下：

1.什么是作用域

按我之前的理解，什么是作用域？就是代码执行时所处区域，那全局的就是在window对象下，局部的就是在全局对象下面或者更下面的。
那书中是怎么说的：

为存储变量以及如何方便找到变量设计的一套规则被称为作用域。

作用域是一套规则？意味わからない、接着往下看…

1.1编译原理

传统编译语言执行之前有三个步骤：

  • 分词/词法分析（Tokenizing/Lexing)
    将由字符组成的字符串分解为对编程语言有意义的代码块，这些代码块被称为词法单元（token)。var a=2, 分解为var、a、=、2、；其中空格是否当作词法单元，取决其是否有意义。
  • 解析/语法分析（parsing）
    将词法单元流（数组）转换为一个程序语法结构树，被称为“抽象语法树”（Abstract Syntax Tree, AST)。var a=2;就有一个VariableBeclaration顶级节点。有一个叫Identifier(a)的子节点，以及AssignmentExpression子节点。AssignmentExpression有一个叫Numricliteral(2)的子节点。
  • 代码生成
    将AST转换为可执行代码的过程。

1.2理解作用域

引擎、编译器、作用域。
编译器将代码编译后给引擎运行，过程中编译器和引擎都会询问作用域。
引擎的查询（LHS，RHS)分别对应变量出现在赋值操作左侧（查容器之后赋值）和非右侧（查值）。

1.3异常

为什么区分LHS和RHS；
变量进行RHS查询时找不到变量会抛出ReferenceError异常；
变量执行LHS查询时找不到变量会在全局创建它（非use strict）;
变量进行对变量进行不合理的操作会抛出TypeError异常；

1.4总结

JS是按照代码块来进行编译和执行的，代码块间相互独立，但变量和方法共享。

1.读入第一个代码块。
2.做分词/语法分析，有错则报语法错误（比如括号不匹配等），并跳转到step5。
3.对var变量和function定义做“预编译处理”（永远不会报错的，因为只解析正确的声明）。
4.执行代码段，有错则报错（比如变量未定义）。
5.如果还有下一个代码块，则读入下一个代码块，重复step2。
6.结束。

2.词法作用域：JS的作用域工作模式是词法作用域。

2.1词法阶段

定义在词法阶段的作用域。

2.2欺骗词法

eval(..)接收一个字符串，视为书写时就存在与程序中这个位置的代码，修改所在词法作用域。

为什么不能预编译，因为字符串内容可以是动态生成的函数代码，词法作用域无法预先确定。
类似的还有setTimeOut(..), setInterval(..), new Function(..)。


严格模式下，eval有其自己的词法作用域。

`with(..)`接收对象创建一个全新的词法作用域，将对象和其属性放进。

严格模式下，with被完全禁止。


编译阶段的性能优化依赖于能够根据代码词法进行静态分析，预先确定所有变量和函数的定义位置，才能快速找到标识符，如果有eval和with, 标识符的位置判断无效，因为不知道作用域会被怎么修改。

3.函数作用域和块作用域

JS执行环境只有两种，全局和局部，环境是编译时就定好的词法作用域，或者eval和with后期修改。

3.1函数中的作用域

隐藏作用域中的变量，最小限度的暴露必要内容。

3.2函数作用域

3.2.1匿名与具名

(function foo(){ .. }) 作为函数表达式意味着foo 只能在.. 所代表的位置中被访问，外部作用域则不行。
函数声明和表达式区分看function关键字出现在生命中的位置。

匿名函数缺点：

  • 调试困难
  • 如何引用自身
  • 可读性

3.2.2立即执行函数

3.3块作用域

块作用域指的是变量和函数不仅可以属于所处的作用域，也可以属于某个代码块（通常指{ .. } 内部）。

  • with
  • try/catch
  • let
  • const

4.提升

这章所提到的内容我还是清楚=.=
总结一下就是在编译后：变量和函数声明会提升到作用域的最上面，函数在变量上面，变量再次声明会被忽略，函数再次声明会被覆盖。

5.作用域闭包

当函数可以记住并访问所在词法作用域时，就产生了闭包。
编译之后形成了词法作用域，本来外面的作用域是访问不到局部作用域中的变量的，那如果局部作用域中的函数在定义时的词法作用域之外的地方被调用，可以继续访问定义时的词法作用域。

5.4循环和闭包

经典的循环打印数字那个题目，所有的setTimeOut的handler全在一个作用域内，这个时候变量已经是跑完的值了, 等到了时间执行，哦豁，变量☞的全是这个值。
解决方法：变量不能指向之前的变量了，那在for的过程中要用另一个变量存储变化值，创建一个局部作用域把存储的变量和setTimeOut包进去，这样每一次log时参数值就不一样了。

5.5模块

模块模式需要两个条件：

  1. 必须有外部封闭函数，至少需要被调用一次；
  2. 封闭函数至少返回一个内部函数，这样才能在私有作用域形成闭包。
