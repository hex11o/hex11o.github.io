Event loop in Node

发表于 2019-04-06 | 阅读次数:

Node的Event Loop分为6个阶段

按照顺序反复运行

timers

timers阶段会执行setTimeout和setInterval回调,并且由poll阶段控制

I/O阶段

处理一些上一轮循环中少数未执行的I/O回调

idle, prepare

阶段内部实现？？？

poll

在这个阶段系统做两件事：

  1. 回到timer阶段执行回调
  2. 执行I/O回调

进入该阶段没有设定timer的话，会发生以下两件事情：

  • 如果poll队列不为空，会遍历回调队列并同步执行，直到队列为空或者达到系统限制
  • 如果poll队列为空
  1. 如果有setImmediate回调需要执行，poll阶段会停止并进入到check阶段执行回调
  2. 如果没有,会等待回调加入到队列中并立即执行回调

check

执行setImmediate

close callbacks

执行close事件

process.nextTick

在每个阶段完成后，如果存在nextTick队列，就会清空队列中所有回调函数，并且优先于其他microtask执行。

Difference between querySelectorAll and getElementBy[]

发表于 2019-03-03 | 分类于 FrontEnd ， JavaScript | 阅读次数:

用querySelectorAll和getElementBy[]时发现它们之间的差别：
querySelectorAll选出的是Nodelist,getElementBy选出来的是HTMLCollection;

  1. Nodelist: NodeList某些情况下是动态的，比如childNodes,进行更新后会反映到NodeList中，其他情况下都是静态的。
  • NodeList的方法：item(),entries(),forEach(),keys(),values()
  • NodeList的属性: length
  1. HTMLCollection: 是类数组的数据，它会根据文档变化进行动态更新，所以你对节点进行遍历操作删减选择器的话,可能出现只操作一半的情况。
  • HTMLCollection的方法: item(),namedItem()
  • HTMLCollection的属性: length

vue+cube-ui构建饿了么

发表于 2019-02-20 | 阅读次数:

搭建脚手架

初始化完成后，项目文件：
.browserslistrc: 不同浏览器的css前缀，给postcss的prefixer插件使用
.eslintrc.js: eslint的配置，插件，配置
.gitignore: git上传时忽略文件
.babel.config.js: 浏览器兼容js
postcss.config.js: postcss的配置
package-lock.json: 版本控制的配置

安装cube-ui

初始化选项有是否后编译，引入接口，定制主体，适配移动rem，vw输出
安装完成，项目添加以及修改文件：

  • src/cube-ui.js: 引入vue,导出模块，vue注册模块
  • src/theme.styl: 配置模块默认样式，主要是颜色
  • vue.config.js: 是一个可选的配置文件，如果项目的根目录存在这个文件，就会被@vue/cli-service自动加载，导出一个包含选项的对象，内容很多，看官方文档去。
  • .browerslistrc: 浏览器同上
  • src/main.js: cube-ui导入
  • package.json,package-lock.json

随后自己修改了vue.config.js的devServer属性，见批注，导入了JSON数据，然后通过before对请求进行了处理，返回了处理后的JSON数据。
