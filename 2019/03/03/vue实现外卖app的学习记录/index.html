vue实现外卖app的学习记录

发表于 2019-03-03 | 分类于 FrontEnd ， 学习记录 | 阅读次数:

学习vue的过程：先看了vue官网文档，初步了解了Vue,Vue-router,Vuex是干什么的，按官方文档写了小实例，然后看慕课网上的vue构建饿了么视频，看实际项目是怎么构建的,先总结一下我初学全家桶的总结，然后说实际构建过程中学到的东西和自己的一些经验。

初入Vue

  • 我学习html,css,js后，将百度技术学院的题做了做，设计到了一些静态页面，在JS里转来转去，终于知道了原型，闭包，了解了一下ES6,ES7，然后做的东西大部分的是通过JS基础实现的一些小需求，后面了解npm, webpack,git，然后过了一遍http的书，准备开始接触vue。

之前不知道什么是框架，那到底什么是框架啊：

  • 简单来说，按这么一套规则来结构化你做的事，提升你编写代码的效率与规范化你的代码，包装好了一些功能，并且能扩展很多功能。

Vue

我理解的Vue, vue.js是一个160KB，4040行的js文件=。=，里面一堆function,我现在是很难看它，等等吧，先知道它在干什么。

start

项目中引用vue.js,就可以使用Vue了，new Vue()创建了一个vue实例,然后传入后面说的el,data,methods,props,watch,钩子函数等等，你可以对这个实例进行一系列指令，所有v-开头的都是Vue的特性，会在渲染DOM上有特定的行为。
vue自带的指令有：v-bind,v-if,v-else,v-else-if,v-show,v-for,v-on,v-model,v-once,v-html,

vue的组件化：每个组件都是有预定义选项的Vue实例。

  1. Vue实例创建时，响应式系统里就加入了data对象中所有属性，这些属性值发生改变，视图就会响应改变。
  2. vue有一些实例属性和方法，都有前缀$,与用户属性进行区分。
  3. 每个实例创建时都会经过一系列过程，这些过程中就可以添加生命周期钩子函数，生命周期：1.beforeCreate,2.created,2.1有el属性吗（没有就等实例的$mount(el)方法）;2.2有template属性吗？render template:el外层做为template 3.beforeMount,创建vm.$el并且把el换掉 4.mounted 5.数据改变前：beforeUpdate 6.虚拟DOM重新渲染：updated 7.beforeDestory($destory()方法调用时) 8.destroyed(把watchers，子组件，时间监听全拆了)。
  4. 计算属性缓存和方法，侦听属性的差别，计算属性的setter
  5. 元素复用, 通过key值属性不同改变
  6. v-if`v-show的差异，v-show`一次渲染，终身切换。
  7. v-for,v-if优先级，v-for的更高，每次v-if都会执行，所以双方结合的实现内容。
  8. 内联语句调用方法，传入参数，特殊参数$event，事件修饰符:.stop,.prevent,.capture,.self,.once,.passive,内联符的顺序问题。
  9. 按键修饰符，kebab-case (page-down)，自定义修饰符别名;系统修饰符.ctrl,.shfit,.alt,.meta(功能键)；.exact修饰符（exactly listed key ）
  10. v-model在,
  • text和textarea元素使用value属性和input事件
  • checkbox和radio使用checked属性和change事件
  • select字段将value作为prop并将change作为事件

深入组件

  1. 自定义事件，自定义事件的触发:$emit('eventName'),可以用第二个参数传入特定值，通过$event访问到抛出的值。
  2. 动态组件，通过:is=“component”绑定组件选项对象，HTML中有些标签对于哪些标签可以出现在内部是有规定的，这时候就不能自定义组件直接往里塞了，要用is来灌入。
  3. 组件命名，kebab-case(my-component),PascalCase(MyComponent),引用和使用。
  4. Prop:Prop就是从标签中传入的数据,父到子是单向下行绑定的，子改变这个属性会发出警告，你需要改变值可以在子组件里再转换一下props(比如data return,或者computed),可以指定类型(可多个)，可选是否必需，设置默认值，自定义验证函数(validator (value){}验证失败会产生警告)
  5. 非Prop的特性，只传向一个组件，但组件并没有prop定义的特性，可以将特性直接添加上去。
  6. style特性和class会自动合并，其他的会被替换掉。
  7. 事件名不存在大小写转换，触发和监听名称需要完全一致，而v-on在DOM模板会被转换为全小写（HTML大小写不敏感），用kebab-case就对了。
  8. 自定义组件v-model的问题，v-model默认利用value的prop和input事件，单选框， 复选框的输控件value值目的可能不同。用model选项避免冲突，就是将有冲突的值写在model属性里。
  9. 组件监听原生事件添加.native修饰符，组件的时间监听可以通过$listners对象保存。
  10. prop双向绑定：子组件触发update:propname事件修改propname属性值，然后父组件监听事件并传入$event值，实现数据跳转，即子组件的——-等会再来——–.sync
  11. 插槽的备用内容，具名插槽：template使用v-slot包裹插槽，模板内用slot name插口，没有包裹的内容默认为default。
  12. 模板作用域问题，模板根节点的数据是在父级作用域中编译的，子模版的所有内容是在子作用域编译的，也就是子模板是无法直接访问父级作用域的数据。
  13. 插槽prop：插槽访问子组件的数据，slot绑定子组件特性，在父级作用域定义提供的插槽prop的名字。
  14. 默认插槽的简写：组件标签直接当作插槽的模板，v-slot用在组件标签上，不能与具名插槽混用。
  15. v-slot:的简写#,一定要带上插槽名字，没有的话就写default。
  16. 动态组件,在具名组件外添加<keep-alive>，保存动态切换的状态，避免反复渲染。
  17. 访问根实例$root和父组件$parent,访问子组件(子组件通过ref赋予name引用，通过$ref.name引用),$ref不是响应式的。
  18. 父组件暴露供后代组件的数据和方法，通过provide属性，在后代组件通过inject属性接收需要的数据和方法。

过渡和动画

  1. 进入/离开的过渡，有六个默认class,v-enter,v-enter-active,v-enter-to,v-leave,v-leave-active,v-leave-to,tansition可以添加name，v切换至相应name。

项目学习记录

数据的配置与调用

  1. 配置mock,配置文件里引入json数据，设置服务器提供的数据

  2. 封装axios,配置api,调用相关api得到服务器提供的相关数据进行处理

动态挂载弹窗

全屏的弹窗适合动态挂载,而不是声明式写死。
cube-ui的createAPI,可以将组件通过该方法会在Vue的prototype上增加一个$create{camelize(component.name)}的方法，这样就可以在其他组件里直接通过这个方法实例化组件，传入的参数有config(Object,组件配置)，renderFn(Function,可选参数,用于生成VNode子节点)，single(Boolean,是否单例)。

config可配置的参数

  1. $props:传递给组件的props。
  2. $events:组件的Events事件回调。

vue实例选项

Vue就是一个构造函数。
构造的时候可以传入哪些参数：

选项/数据

  1. data: 生成实例时的数据对象，会将data的属性转换为getter/setter,从而能双向响应，通过vm.$data访问到原始数据，组件定义时，申明为一个返回函数，避免数据共享.
  2. props: 用来接收父组件的数据。
  3. propsData: 只是方便测试向用new创建的实例传递props
  4. computed： 经过计算之后的属性对象。
  5. methods：通过实例能访问的方法。
  6. watch: 实例生成时，立即调用$watch(),遍历watch对象的每一个属性，发生变化时立即执行回调函数，或方法名，或含选项的对象。

选项/DOM

  1. el: 挂载在DOM元素，后续需要挂的话用vm.$mount()挂载。
  2. template: vue实例的结构模板，如果标签中间还有内容，可以用插槽<slot>将其置入。
  3. render: 接收一个createElement方法作为第一个参数来创建VNode,这里是建立虚拟DOM,。
  4. renderErr: 开发环境使用，render函数不行时，提供另一种输出，render失败返回的对象作为第二个参数传入renderErr。

选项/生命钩子

  1. beforeCreate：实例初始化后，观测(data observer)，配置(event/watcher)之前调用。
  2. created: 实例创建完成,观测，运算，配置完成，还没挂载。
  3. beforeMount: 实例挂载之前，render函数调用，生成虚拟DOM，服务器端不被调用.
  4. mounted: el被vm.$el替换，挂载到实例后调用，子组件可以往后稍稍，如果真需要视图全部渲染，用this.$nextTick()替换
  5. beforeUpdate: 发生在虚拟DOM打补丁之前，适合更新前访问现在的DOM,比如移除添加的事件监听。
  6. updated: 数据更改后虚拟DOM重新渲染之后。
  7. actived: keep-alive组件激活时调用，动态组件：切换不重复渲染。
  8. deactived: keep-alive组件停用时调用。
  9. beforeDestory: 实例销毁之前调用.
  10. destoryed: 实例销毁之后调用。
  11. errorCaptured: 子孙组件发生错误时调用，接收三个参数：错误对象，发生错误的组件，错误来源信息。

选项/资源

  1. directive: 可用指令的hash表。
  2. filtes: 可用过滤器的hash表。
  3. components: 可用组件的哈希表。

选项/组合

  1. parent: 实例的父实例，this.$parent访问父实例。
  2. mixins: array
  3. extends: Object|Function, 扩展另一个组件。
  4. provide/inject: provide是一个对象或返回对象的函数，包含给子孙使用的属性,inject是provide提供的属性名或属性数组

选项/其他

  1. name：模板自身递归调用？？那不是炸刚了么。
  2. delimiters: 改变纯文本插入分隔符：[‘${‘,’}’]
  3. functional: 使组件无data和无this.
  4. model：使v-model时定制prop和event.这里看不懂
  5. inheritAttrs: 未被作为props给子组件的特性，设置false会将其去掉，否则会作为普通attrs应用到子组件根元素上。
  6. comments: 设置为true时，保留渲染模板中的HTML注释，默认舍弃。

实例/属性

  1. $data
  2. $props
  3. $el
  4. $options: 当前实例的初始化选项
  5. $parents: 父组件
  6. $root: 根组件
  7. $children: 直接子组件，数组
  8. $slots: 插槽的内容，具名插槽有对应属性，default属性有所有匿名插槽。
  9. $scopedSlots: 访问作用域插槽。
  10. $refs: 注册过ref特性的所有DOM元素和组件实例。
  11. $attrs: 父作用域中不作为prop被是被的特性绑定，除开class和style。
  12. $listeners: 父作用域中的@事件监听器对象。

实例方法/数据

  1. $watch(exporfn,callback,{options}):第一个参数为键路径或计算属性的函数，第二个参数为回调函数，第三个参数为对象选项：deep,immediate
  2. $set(target,key,value)
  3. $delete(target,key)

实例方法/事件

  1. $on(event,callback)
  2. $one(event,callback):自定义事件，只触发一次。
  3. $off([event,callback])：事件和函数双向组合移除，没有参数全部移除。
  4. $emit(eventName,[..args])

实例生命周期

  1. $mount()
  2. $forceUpdate()
  3. $nextTick([callback]):下次DOM更新
  4. $destory()

指令

v-text,v-html,v-show,v-if,v-else,v-else-if,v-for,v-on,v-bind,v-model,v-slot,v-pre,v-cloak,v-once

特殊特性

  1. key: 新旧对比辨识VNodes.
  2. ref: 用来给元素或子组件注册引用信息，会注册在父组件的$refs对象上。
  3. is：动态组件基于DOM内模板限制来工作。

内置组件

  1. component
  2. transition
  3. transition-group: 作为多个元素/组件的过渡效果，默认渲染span,可以通过tag属性配置哪个元素应该被渲染，子节点必需有独立的key。
  4. keep-alive: 会缓存不活动的组件实例，而不是销毁他们，设置include，exclude,max设置缓存条件。
  5. slot： 作为组件模板之中的内容分发插槽。

全局config

  1. silent:boolean 取消所有日志和警告
  2. optionMergeStrategies: 自定义合并策略的选项。
  3. keyCodes： 自定义键位别名

全局API

  1. extend(options): 使用构造器，创建一个包含组件选项的对象构造器。
  2. nextTick([callback,context]): 下次更新DOM结束后执行延迟回调。
  3. set
  4. delete
  5. directive：自定义指令，自定义指令的生命钩子(bind, inserted, update, componentUpdated, unbind)
  6. filter(‘id’,[definition]): 注册or获取全局过滤器
  7. component
  8. use
  9. mixin
  10. observable
